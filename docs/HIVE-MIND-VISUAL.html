Wiki Compiler
LLM-powered synthesis · incremental compilation · markdown export
Entity Pages
One page per knowledge graph entity. Aggregates related memories, relations, and temporal history.
  • Person, project, concept, organization, technology, tool, location, event
  • Cross-linked via entity relations
Concept Pages
Synthesized knowledge around recurring themes and concepts across your memory.
  • LLM-generated narrative summaries
  • Pattern detection across frames
Synthesis Pages
High-level overviews combining entities and concepts into comprehensive narratives.
  • Cross-domain synthesis
  • Automatic table of contents
Index Page
Auto-generated navigation hub listing all compiled pages by type.
Incremental Compilation
Watermark-based tracking recompiles only changed pages. Avoids redundant LLM calls.
Health Reports
Detects contradictions, knowledge gaps, orphan entities, and weak-confidence pages.
  • Data quality score
  • Actionable remediation hints
LLM Providers
OpenAI, Anthropic, or Ollama for synthesis. Auto-resolves from environment.
Markdown Export
Standard markdown output. Compatible with any viewer, SSG, or documentation tool.
3
Memory Substrate
6 subsystems · hybrid search · knowledge graph · temporal validity
FrameStore
Video-codec-inspired memory frames with importance scoring and content deduplication.
  • I-frames: immutable facts (anchors)
  • P-frames: procedural knowledge (deltas)
  • B-frames: bidirectional references
  • 5 importance levels: critical to deprecated
  • Content-hash deduplication
HybridSearch
Dual-mode search fused via Reciprocal Rank Fusion for maximum recall.
  • FTS5 keyword search (porter + unicode61)
  • sqlite-vec vector search
  • RRF fusion (k=60)
  • Scoring profiles: balanced, recent, comprehensive
  • Recency, frequency, importance weighting
KnowledgeGraph
Typed entities and relations with bitemporal validity and confidence scoring.
  • 8 entity types
  • Directed, typed relations
  • Temporal validity (valid_from / valid_to)
  • Confidence scoring (0.0 - 1.0)
  • Bitemporal tracking
IdentityLayer
Persistent user profile. Name, role, personality, capabilities. Survives all sessions.
AwarenessLayer
Active task and context tracking with TTL-based expiry. Maintains situational awareness.
  • Categories: task, action, pending, flag
  • Priority ordering
  • Auto-expiry via TTL
SessionStore
Conversation tracking with project mapping. Daily auto-grouping for MCP sessions.
  • GOP ID to project mapping
  • Active / closed / archived states
  • Summary field for session context

5
Storage Layer
SQLite + sqlite-vec · WAL mode · per-workspace isolation
SQLite (better-sqlite3)
Synchronous, high-performance embedded database. WAL mode for concurrent reads.
  • No external server required
  • Write-Ahead Logging
  • ACID transactions
sqlite-vec Extension
Vector similarity search inside SQLite. Cosine similarity via vec0 virtual tables.
  • No separate vector DB
  • Embedding storage + query
  • Cosine similarity
FTS5 Indexes
Full-text search with porter stemming. Unicode-aware tokenization.
Per-Workspace Isolation
Each workspace owns a separate .mind database file. Complete data isolation.
  • personal.mind + workspace.mind
  • Created on demand
  • Independent schema lifecycle
Auto-Init
Zero configuration. Auto-creates ~/.hive-mind/ and initializes schema on first run.
Schema Versioning
Meta table tracks schema version. Forward migration on upgrade.
